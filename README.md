Готовый серверный конфиг [Xray](https://github.com/XTLS/Xray-core) для создания туннеля, который выглядит как **легитимный DNS‑over‑TCP‑сеанс** на 53‑м порту.  
Разработан для скрытой передачи трафика там, где DPI разрешает DNS, но блокирует VPN/прокси.

---

## ✨ Особенности

- 🎯 **Порт 53 (DNS)** — стандартный порт для DNS‑over‑TCP, не вызывает подозрений.
- 📨 **Первый пакет — DNS‑ответ** — сервер при подключении отправляет бинарный пакет, который декодируется в ответ `yandex.ru → 77.88.8.8`.
- 🕵️ **FakeDNS** — все реальные DNS‑запросы клиента оборачиваются внутрь зашифрованного туннеля; снаружи ни одной утечки имён.
- 🚫 **Блокировка торрентов** — встроенное правило, чтобы не «светить» P2P-трафик.
- ⏱️ **Тонкая настройка таймаутов** — `policy` ограничивает время рукопожатия и размеры буфера, имитируя поведение обычного DNS‑клиента.

---

## 🔍 Принцип маскировки (что видит DPI)

1. **SYN → SYN‑ACK → ACK** — обычное TCP‑рукопожатие на 53‑й порт.
2. **Сервер шлёт первый пакет** — это бинарный DNS‑ответ:  
   `yandex.ru → 77.88.8.8` (A‑запись, упакованная в HEX).
3. DPI видит валидный DNS‑over‑TCP‑ответ и классифицирует поток как **легитимный DNS**.
4. Дальнейший обмен — шифрованные VLESS‑данные — для инспектора выглядит как продолжение DNS‑сессии (дополнительные запросы/ответы).

Таким образом, даже продвинутые системы, ориентированные на сигнатуры протоколов, считают соединение «чистым DNS».
